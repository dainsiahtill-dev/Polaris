# Engineering Hygiene Remediation Blueprint

**Date**: 2026-04-24
**Status**: Active
**Scope**: 全仓工程素养整改 — 文档一致性、术语规范化、目录治理、测试补全

---

## 1. 问题背景

外部审计（2026-04-24）发现三个系统性问题：

| 问题 | 严重度 | 具体表现 |
|------|--------|----------|
| 术语泛滥 | 🔴 高 | 三套隐喻系统叠加，同一实体 2-3 个别名 |
| 目录混乱 | 🟠 中高 | 11 层嵌套、121 个 blueprint 无索引、命名不统一 |
| 数据自相矛盾 | 🔴 高 | Cell 数 51 vs 59、Python 文件 1642 vs 2732、CLI 入口失效 |

## 2. 架构目标

```

```

## 4. 核心数据流

```
外部审计报告

    ├──► Team 1 ──► 修复文档矛盾 ──► 验证三份文档数据一致

    ├──► Team 5 ──► 补全关键测试 ──► 覆盖率提升
    └──► Team 6 ──► 激活治理规则 ──► enforce 比例 → 60%+
```

## 5. 技术选型理由

- **术语表用 Markdown**：与现有文档体系一致，便于 Agent 读取
- **蓝图索引用 Markdown**：保持 `docs/blueprints/` 自包含
- **测试用 pytest**：与现有测试框架一致
- **治理规则用 YAML**：与现有 `fitness-rules.yaml` 一致

## 6. 验收标准

| 维度 | 验收指标 |
|------|----------|
