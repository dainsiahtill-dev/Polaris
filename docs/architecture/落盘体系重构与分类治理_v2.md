# Polaris 落盘体系重构与分类治理

**版本**: v2.0  
**状态**: 已实现  
**日期**: 2026-03-12

---

## 1. 设计目标

将 Polaris 落盘体系重构为「当前态与历史态解耦、路径契约统一、一次切换可回滚」：

1. **当前态真相源**：`runtime/tasks/` 保持不变，作为实时任务状态来源
2. **历史累计归档**：`workspace/history/` 统一归档，支持检索 manifest 与索引
3. **一次切换**：通过迁移脚本一次性切换，支持回滚
4. **零信任**：任何上游输出都不默认可信，必须二次校验

## 2. 存储分类体系

### 2.1 逻辑路径分类

所有路径分为以下类别：

| 类别 | 逻辑路径前缀 | 物理路径 | 生命周期 | 归档策略 |
|------|-------------|----------|----------|----------|
| **全局配置** | `config/*` | `~/.polaris/config/*` | permanent | 不归档 |
| **工作区持久** | `workspace/docs/*` | `<workspace>/.polaris/*` | permanent | 不归档 |
| **工作区持久** | `workspace/brain/*` | `<workspace>/.polaris/*` | permanent | 不归档 |
| **工作区持久** | `workspace/policy/*` | `<workspace>/.polaris/*` | permanent | 不归档 |
| **工作区持久** | `workspace/meta/*` | `<workspace>/.polaris/*` | permanent | 不归档 |
| **工厂当前态** | `workspace/factory/*` | `<workspace>/.polaris/factory/*` | active | 终态归档 |
| **运行时当前态** | `runtime/contracts/*` | `<runtime_root>/*` | active | 终态归档 |
| **运行时当前态** | `runtime/tasks/*` | `<runtime_root>/tasks/*` | active | 快照归档 |
| **运行时当前态** | `runtime/state/*` | `<runtime_root>/state/*` | ephemeral | 窗口保留 |
| **运行时当前态** | `runtime/events/*` | `<runtime_root>/events/*` | active | 终态压缩归档 |
| **运行时快照** | `runtime/runs/<run_id>/*` | `<runtime_root>/runs/*` | run-scoped | 终态归档 |
| **历史归档** | `workspace/history/*` | `<workspace>/.polaris/history/*` | history | 永久保留 |

### 2.2 生命周期模型

```
┌─────────────────────────────────────────────────────────────────┐
│                        生命周期流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  PERMANENT ─────────────────────────────────────────────────►   │
│  (永久保留)                                                      │
│                                                                 │
│  ACTIVE ──────────────────────► 终态 ──► HISTORY (归档)        │
│  (当前活跃)     ↑                      │                         │
│       │        │                      │                         │
│       └────────┴──────────────────────┘                         │
│                                                                 │
│  EPHEMERAL ────────────────────► 清理                           │
│  (短期临时)                                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 核心模块

### 3.1 存储策略层 (`storage_policy.py`)

**文件**: `src/backend/core/polaris_loop/storage_policy.py`

定义存储分类、生命周期和保留策略。

```python
# 存储分类
class StorageCategory(Enum):
    GLOBAL_CONFIG = "global_config"
    WORKSPACE_PERSISTENT = "workspace_persistent"
    RUNTIME_CURRENT = "runtime_current"
    RUNTIME_RUN = "runtime_run"
    WORKSPACE_HISTORY = "workspace_history"
    FACTORY_CURRENT = "factory_current"
    FACTORY_HISTORY = "factory_history"

# 生命周期
class Lifecycle(Enum):
    PERMANENT = "permanent"    # 永久保留
    ACTIVE = "active"          # 当前活跃，可覆盖
    EPHEMERAL = "ephemeral"   # 短期临时，可清理
    HISTORY = "history"        # 历史归档，只读压缩

# 存储策略
@dataclass(frozen=True)
class StoragePolicy:
    logical_prefix: str
    category: StorageCategory
    lifecycle: Lifecycle
    retention_days: int = -1      # -1 表示永久
    compress: bool = False        # 归档时是否压缩
    archive_on_terminal: bool = False  # 终态时是否归档
```

### 3.2 历史归档服务 (`history_archive_service.py`)

**文件**: `src/backend/app/services/history_archive_service.py`

提供统一的归档操作接口。

```python
class HistoryArchiveService:
    def __init__(self, workspace: str):
        # 解析存储根路径
        from core.polaris_loop.storage_layout import resolve_storage_roots
        self._storage_roots = resolve_storage_roots(workspace)
        self.history_root = Path(self._storage_roots.history_root)
        self.runtime_root = Path(self._storage_roots.runtime_root)

    # 归档运行
    def archive_run(self, run_id: str, reason: str, status: str) -> ArchiveManifest:
        """
        归档 runtime/runs/<run_id> 到 workspace/.polaris/history/runs/<run_id>
        - 复制整个运行目录
        - 压缩事件文件 (*.jsonl -> *.jsonl.zst)
        - 生成 manifest.json
        - 更新 runs.index.jsonl
        """
        
    # 归档任务快照
    def archive_task_snapshot(self, snapshot_id: str, ...) -> ArchiveManifest:
        """
        归档 runtime/tasks/ 到 workspace/.polaris/history/tasks/<snapshot_id>
        - snapshot_id 格式: pm-{iteration:05d}-{timestamp}
        """
        
    # 归档工厂运行
    def archive_factory_run(self, factory_run_id: str, ...) -> ArchiveManifest:
        """
        归档 workspace/.polaris/factory/<run_id> 
        到 workspace/.polaris/history/factory/<run_id>
        """
```

### 3.3 归档钩子 (`archive_hook.py`)

```python
class ArchiveHook:
    def trigger_run_archive(self, run_id: str, reason: str, status: str) -> None:
        """异步触发运行归档，立即返回"""
        asyncio.create_task(self._archive_run_async(run_id, reason, status))
        
    def trigger_task_snapshot_archive(self, snapshot_id: str, ...) -> None:
        """异步触发任务快照归档，立即返回"""
        
    def trigger_factory_archive(self, factory_run_id: str, ...) -> None:
        """异步触发工厂归档，立即返回"""
```

## 4. 归档触发点

### 4.1 Workflow 终态归档

**文件**: `src/backend/core/orchestration/orchestration_service_impl.py`

```python
async def _execute_workflow(self, run_id: str, ...):
    try:
        # ... 执行逻辑 ...
        
        # 设置终态
        if failed_tasks:
            snapshot.status = RunStatus.FAILED
        else:
            snapshot.status = RunStatus.COMPLETED
            
        await self._update_snapshot(snapshot)
        
        # 触发归档 (非阻塞)
        self._trigger_run_archive(run_id, snapshot.status)
        
    except asyncio.CancelledError:
        snapshot.status = RunStatus.CANCELLED
        await self._update_snapshot(snapshot)
        self._trigger_run_archive(run_id, RunStatus.CANCELLED)
        raise
```

**触发条件**: COMPLETED / FAILED / CANCELLED / BLOCKED / TIMEOUT

```python
def _finalize_iteration(self, workspace_full: str, iteration: int, stage: str):
    """PM 迭代完成时触发归档"""
    
    # 生成 snapshot_id
    snapshot_timestamp = datetime.now().strftime("%Y%m%d%H%M%S")
    snapshot_id = f"pm-{iteration:05d}-{snapshot_timestamp}"
    
    # 异步触发归档 (非阻塞)
    if os.path.isdir(tasks_dir):
        hook = get_archive_hook(workspace_full)
        hook.trigger_task_snapshot_archive(
            snapshot_id=snapshot_id,
            source_tasks_dir=tasks_dir,
            source_plan_path=plan_path if os.path.isfile(plan_path) else None,
            reason=stage,
        )
```

**触发条件**: PM 迭代完成（stage = "pm_completed" / "pm_failed" 等）

```python
def update_status(self, task_id: int, status: TaskStatus, ...):
    """任务状态变更时记录事件"""
    
    with self.transaction():
        # ... 更新任务状态 ...

    if terminal_event_data:
        self._write_terminal_event_async(terminal_event_data)

def _write_terminal_event_async(self, event_data: Dict[str, Any]):
    """后台线程写入事件，不阻塞主流程"""
    
    def _write():
        events_dir.mkdir(parents=True, exist_ok=True)
        with open(event_file, "a", encoding="utf-8") as f:
            json.dump(event_data, f, ensure_ascii=False)
            f.write("\n")
            f.flush()
            os.fsync(f.fileno())
    
    thread = threading.Thread(target=_write, daemon=True)
    thread.start()
```

## 6. 迁移脚本

**文件**: `src/backend/scripts/migrate_storage_layout_v2.py`

### 6.1 使用方法

```bash
# 预演 (不实际执行)
python migrate_storage_layout_v2.py --workspace /path/to/workspace --dry-run

# 执行迁移
python migrate_storage_layout_v2.py --workspace /path/to/workspace

# 严格模式 (默认)
python migrate_storage_layout_v2.py --workspace /path/to/workspace --strict-cutover

# 验证
python migrate_storage_layout_v2.py --workspace /path/to/workspace --verify

# 回滚
python migrate_storage_layout_v2.py --workspace /path/to/workspace --rollback
```

### 6.2 执行流程

```
1. 预检
   ├── 校验 runtime 和 history 目录可写
   └── 检查迁移版本文件

2. 备份
   ├── 备份 runtime_root 到 migration_backup_<timestamp>
   └── 备份 workspace/.polaris/factory

3. 迁移
   ├── runtime/runs/* -> workspace/.polaris/history/runs/*
   ├── workspace/.polaris/factory/* -> workspace/.polaris/history/factory/*
   └── 规范化 legacy 命名

4. 索引重建
   ├── 生成 runs.index.jsonl
   └── 生成 factory.index.jsonl

5. 切换
   ├── 写入 storage_layout.version.json (version=2)
   └── 设置 legacy_read_fallback=false

6. 验证
   ├── 校验文件数量一致
   ├── 校验 hash 一致
   └── 校验索引可查询
```

## 7. 数据结构

### 7.1 ArchiveManifest

```python
@dataclass
class ArchiveManifest:
    scope: str                    # "run" | "task_snapshot" | "factory_run"
    id: str                       # run_id | snapshot_id | factory_run_id
    archive_timestamp: float       # Unix 时间戳
    archive_datetime: str          # ISO 格式
    source_runtime_root: str       # 源 runtime 根路径

    file_count: int               # 文件数量

    reason: str                   # 归档原因
    compressed: bool               # 是否压缩
    compression_ratio: float        # 压缩率
```

## 9. 压缩支持

### 9.1 Zstandard 压缩

归档时自动压缩事件文件：

```python
# *.jsonl -> *.jsonl.zst
with open(jsonl_file, "rb") as f_in:
    with open(compressed_path, "wb") as f_out:
        compressor = zstd.ZstdCompressor()
        compressor.copy_stream(f_in, f_out)
```

### 9.2 自动解压缩

读取时自动解压缩：

```python
def get_run_events(self, run_id: str) -> List[Dict]:
    # 优先读取压缩版本
    if compressed_path.exists() and ZSTD_AVAILABLE:
        with open(compressed_path, "rb") as f:
            dctx = zstd.ZstdDecompressor()
            with dctx.stream_reader(f) as reader:
                for line in reader:
                    yield json.loads(line)
```

## 10. 错误处理

### 10.1 归档失败策略

```
┌─────────────────────────────────────────────────────────────┐
│                      归档失败处理                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  触发归档                                                    │
│      │                                                      │
│      ▼                                                      │
│  ┌───────────────┐                                         │
│  │ 归档成功?     │──Yes──► 更新索引，记录日志              │
│  └───────┬───────┘                                         │
│      No  │                                                 │
│      ▼   │                                                 │
│  ┌───────────────┐                                         │
│  │ 记录错误日志   │                                         │
│  │ (不阻断主流程) │                                         │
│  └───────────────┘                                         │
│      │                                                      │
│      ▼                                                      │
│  可选: 加入重试队列                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 10.2 异常捕获

所有归档调用都使用 try-except 包装：

```python
try:
    hook.trigger_run_archive(run_id, reason, status)
except Exception as e:
    logger.warning(f"Failed to trigger archive: {e}")
    # 不阻断主流程
```
