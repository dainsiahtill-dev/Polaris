## 压测目标

你要验证的不是“某个项目能不能偶然做出来”，而是 Polaris 是否能在不同类型、不同复杂度、不同技术约束下，持续稳定地产出可构建、可测试、可验收的项目。

压测必须覆盖：

- CRUD / 表单 / 数据管理
- 实时通信 / 同步
- 编辑器 / 内容处理
- 工具型应用 / 自动化
- 安全 / 加密 / 文件处理
- 互动 / 游戏 / 计时类 UI

## 轮次规则

1. 不得连续两轮都只做同一类型的简单 CRUD 项目。
2. 每轮项目必须满足 `v5.1` 中的复杂度门槛，并额外具备至少 2 个增强特性。
3. 如果重复压测同一个项目主题，下一轮必须显式升阶，例如：
   - 增加持久化
   - 增加用户配置或环境变量
   - 增加 WebSocket / SSE / 实时状态
   - 增加导入导出
   - 增加权限、加密或审计日志
   - 增加单元测试 + 集成测试
   - 增加构建脚本 / 发布脚本
4. 每轮都必须记录：
   - 目标项目名称
   - 项目类别
   - 使用的增强特性
   - Polaris 失败点
   - 修复动作
   - 回归结果

## 选题策略

推荐轮换顺序：

1. 先做 CRUD / 数据管理类
2. 再做实时 / WebSocket 类
3. 再做编辑器 / 内容处理类
4. 再做安全 / 文件类
5. 再做游戏 / 互动类
6. 然后回到更高复杂度的 CRUD 或 CMS

如果要跑长周期压测，优先保证题材分布均衡，而不是只挑容易成功的项目。

## 项目池

每个项目至少满足“核心能力 + 增强特性 + 可测验收”。

1. 个人记账簿（账单管理）
   - 核心能力：账单录入、分类、筛选、月度汇总
   - 建议增强：预算预警、图表统计、CSV 导入导出
   - 压测重点：表单状态、持久化、一致性校验

2. 待办事项清单（To-Do List）
   - 核心能力：新增、编辑、完成、优先级、标签
   - 建议增强：拖拽排序、分组视图、提醒、离线缓存
   - 压测重点：状态流转、过滤组合、交互回归

3. 简易 Markdown 编辑器
   - 核心能力：编辑、实时预览、文档保存
   - 建议增强：目录导航、语法高亮、导出 HTML / PDF
   - 压测重点：文本处理、预览同步、内容渲染安全

4. 实时聊天室（WebSocket）
   - 核心能力：多人消息、在线状态、房间切换
   - 建议增强：消息历史、输入中提示、重连策略
   - 压测重点：实时连接、状态同步、异常恢复

5. 博客系统（CMS）
   - 核心能力：文章创建、编辑、发布、列表展示
   - 建议增强：草稿/发布状态、分类标签、搜索、管理后台
   - 压测重点：内容模型、路由、权限边界

6. 天气预报展示器
   - 核心能力：城市查询、天气卡片、多日预报
   - 建议增强：最近搜索、定位、缓存、错误降级
   - 压测重点：第三方 API 适配、缓存、降级策略

7. 个人简历生成器
   - 核心能力：表单录入、简历模板、导出
   - 建议增强：多模板切换、主题配置、PDF 导出
   - 压测重点：模板渲染、结构化数据、导出质量

8. 抽奖 / 随机点名工具
   - 核心能力：名单导入、随机选择、结果展示
   - 建议增强：去重规则、权重设置、历史记录
   - 压测重点：随机逻辑、状态控制、交互动效

9. 番茄钟（专注计时器）
   - 核心能力：专注/休息倒计时、阶段切换、记录统计
   - 建议增强：通知、声音提醒、周期配置
   - 压测重点：计时精度、前后台状态一致性、持久化

10. 密码管理器（加密存储）
    - 核心能力：密码条目管理、本地加密、解锁查看
    - 建议增强：主密码、分类、复制保护、强度提示
    - 压测重点：加密边界、敏感数据处理、防误泄漏

11. 图片占位符生成器
    - 核心能力：自定义尺寸、文字、背景色、下载
    - 建议增强：批量生成、预设模板、URL 参数化
    - 压测重点：参数验证、图像生成、批处理流程

12. 在线剪贴板（跨端传词）
    - 核心能力：文本发送、接收、短期存储
    - 建议增强：过期时间、多设备同步、历史记录
    - 压测重点：同步一致性、权限控制、临时数据清理

13. 聚合搜索工具（一键搜多站）
    - 核心能力：统一输入、多个站点跳转或聚合结果
    - 建议增强：搜索模板管理、快捷键、历史记录
    - 压测重点：配置化、跳转逻辑、结果整合

14. 简易单位转换器（汇率 / 度量）
    - 核心能力：基础单位换算、双向输入、结果展示
    - 建议增强：汇率缓存、常用组合、最近记录
    - 压测重点：计算正确性、配置扩展、边界值测试

15. 文件断点续传器
    - 核心能力：分片上传、断点恢复、进度展示
    - 建议增强：校验和、并发上传、失败重试
    - 压测重点：文件处理、恢复逻辑、异常注入

16. 静态网站生成器（SSG）
    - 核心能力：Markdown 输入、模板渲染、静态输出
    - 建议增强：导航生成、分页、标签页、构建命令
    - 压测重点：内容编译链、文件系统、构建产物校验

17. RSS 阅读器
    - 核心能力：订阅源管理、文章列表、已读状态
    - 建议增强：抓取缓存、关键词过滤、收藏
    - 压测重点：抓取兼容性、解析容错、状态持久化

18. 自动化签到脚本
    - 核心能力：目标站点登录、签到执行、结果记录
    - 建议增强：定时任务、失败告警、多站点配置
    - 压测重点：自动化稳定性、重试、凭据管理

19. 屏幕截图 / 录屏工具
    - 核心能力：截图、区域选择、文件保存
    - 建议增强：录屏、快捷键、历史记录、格式配置
    - 压测重点：桌面能力、文件输出、性能与权限

20. 贪吃蛇 / 俄罗斯方块小游戏
    - 核心能力：可操作游戏循环、得分、重新开始
    - 建议增强：难度、排行榜、本地存档、音效
    - 压测重点：渲染刷新、状态机、输入响应

## 每轮必须追加的增强特性候选

每轮至少选择 2 项；如果是高强度压测，选择 3 到 4 项：

- 本地持久化
- 导入导出
- WebSocket / SSE 实时更新
- 用户配置或环境变量
- 单元测试
- 集成测试
- 错误处理与回退机制
- 构建脚本 / 发布脚本
- 审计日志
- 权限或加密
- 批处理能力
- 离线缓存

## 禁止事项

- 不得把“做个静态页面”伪装成完整项目压测
- 不得因为为了通过而删掉必要功能
- 不得连续多轮都只选择最容易实现的题材
- 不得手工修改目标项目代码来掩盖 Polaris 的执行缺陷
- 不得用旧字段、旧接口、旧兼容层来绕过当前 Polaris 正式合同
