# Polaris 元设计师自动化测试提示词（v5.0 - 当前对齐别名）

本文件是 `v5.1` 的历史别名入口，内容已经按当前 Polaris 项目事实对齐。
它不代表继续兼容旧接口、旧字段或旧架构分层。
如需完整执行合同，以 `docs/prompt/元设计师-自动化测试v5.1.md` 为准。

## 当前必须接受的事实

1. 主执行栈：`Playwright -> Computer Use`
2. 当前 AGI 运行内核是 `resident`，执行仍经 `PM/Director`
3. AGI 没有独立 LLM role；以 `pm/director` 配置为准
4. `workspace` 必须同时在设置、环境变量和全局持久化中保持一致
5. Core 层 LLM 访问只走 `ServiceLocator`，不能重新引入 `app.llm.*` 直接依赖
6. 编排共享合同正式位于 `src/backend/core/orchestration/`
7. 对外 workflow/runtime 合同统一使用 `id`
8. 对外超时与重试字段统一使用 `timeout_seconds`、`global_timeout_seconds`、`initial_interval_seconds`、`max_interval_seconds`
9. Director 任务血缘正式证据字段仍是 `metadata.pm_task_id`
10. 只能修改 Polaris，不能修改目标项目代码

## 当前强制流程

1. 环境预检
2. Workspace / 持久化门禁
3. Docs Wizard / Architect project initiation
4. PM 质量门禁
5. Director 工具审计
6. QA 验收
7. AGI / Resident 观察与事实校验
8. 修复循环直到通过

## 必过门禁

- PM 分数 >= 80
- critical issues = 0
- Director 任务必须至少有一条具备 `metadata.pm_task_id`
- `integration_qa_passed` 或明确失败原因
- workspace 重启后不能回退
- 提示词穿透命中为 P0

## 代码收敛要求

- 涉及 core 层 LLM 修复时，只能维持 `core -> ServiceLocator -> app.llm.providers` 注册链
- 涉及共享编排合同、UI 状态、文件追踪时，只能落在 `src/backend/core/orchestration/`
- 涉及外部 JSON / runtime contract 时，只能使用 `id`、`timeout_seconds`、`global_timeout_seconds`、`initial_interval_seconds`、`max_interval_seconds`
- `task_id` 仅允许作为 Python 内部属性名存在，不能作为对外 contract 旧键继续扩散
- 旧键 `timeout` / `global_timeout` / 旧 retry 别名视为漂移，不能再通过兼容层兜底
