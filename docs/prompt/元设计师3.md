# Polaris 元设计师提示词（检查清单版）

1. 启动 Polaris
2. 获取 backend info
3. 校验 `/settings`、`/runtime/storage-layout`、`/llm/status`
4. 校验 workspace 正确且可持久化
5. 若未开启自我升级模式，确认目标 workspace 不在 Polaris 元项目根目录或其子目录内
6. Complete Docs Wizard / Architect project initiation
7. 驱动 PM，检查质量分 >= 80 且无 prompt leakage
8. 驱动 Director，检查 `RUNNING` 状态与 `metadata.pm_task_id` 血缘
9. 检查外部 contract 是否仍使用 `id`、`timeout_seconds`、`global_timeout_seconds`、`initial_interval_seconds`、`max_interval_seconds`
10. 检查 QA 结果
11. 检查 AGI / Resident 的 `active_workspace` 与事实叙事
12. 失败则修 Polaris，不能改目标项目代码
13. 回归失败门禁
14. 回归整链

注意：

- 当前 AGI 不等于独立 LLM role
- 当前主执行栈是 `Playwright -> Computer Use`
- workspace 回退是 P0
- 未开启自我升级模式却把 workspace 指到 Polaris 元项目视为 P0
- Core 层角色 LLM 访问只走 `ServiceLocator`
- 编排共享合同正式位于 `src/backend/core/orchestration/`
- 旧键 `task_id`、`timeout`、`global_timeout` 与旧 retry 别名视为漂移，不得再靠兼容层兜底
- 完整合同见：`docs/prompt/元设计师-自动化测试v5.1.md`
