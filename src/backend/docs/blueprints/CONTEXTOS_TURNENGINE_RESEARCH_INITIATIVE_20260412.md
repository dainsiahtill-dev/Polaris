## 3. 关键审计议题

### 议题1: 事件溯源完整性 (ContextOS)

**现状疑点**:
```python
# 发现的问题
tool_call    iter=None  # ❌ 缺失
llm_call_end iter=14    # ✅ 正常
```

**审计问题**:
1. 为什么Tool Call的iteration为None？
2. 事件序列号轮转是否会导致数据丢失？
3. 如何确保跨Run的事件可追溯性？

**研究目标**:
- 设计完整的事件因果链追踪
- 评估序列号64位整数溢出风险

### 议题2: 断路器系统有效性 (TurnEngine)

**现状疑点**:
- 日志中未找到Circuit Breaker事件
- 但代码已实现断路器逻辑

**审计问题**:
1. 断路器是否真正阻止了死循环？
2. 阈值参数(3次重复/5次只读)是否最优？
3. 恢复状态机是否有效运转？

**研究目标**:
- 建立断路器效果量化指标
- 设计自适应阈值算法

### 议题3: 上下文压缩质量

**现状疑点**:
- 紧急压缩在50事件时触发
- 但未测量信息保留率

**审计问题**:
1. Turn-Block压缩是否保留完整工具链？
2. 压缩后的提示词质量如何评估？
3. 是否存在"压缩导致的幻觉"？

**研究目标**:
- 建立压缩质量评估框架
- 设计人类可读的压缩报告

### 议题4: 意图切换语义连续性

**现状疑点**:
- 用户从"查看"切换到"创建"
- 但旧分析内容仍可能占用上下文

**审计问题**:
1. 意图切换检测是否准确？
2. 摘要提取是否保留关键信息？
3. 是否存在"意图切换导致的遗忘"？

**研究目标**:
- 设计语义连续性评估基准
- 建立用户满意度反馈机制
